# Tailwind, Theming & UI

## Tailwind v4

- Tailwind is loaded via `@tailwindcss/postcss` and `@import "tailwindcss"` in
  `app/globals.css`. There is **no `tailwind.config.js`** — design tokens live
  in CSS via `@theme inline { … }` and `:root` / `.dark` blocks.
- Always merge incoming class props with `cn(…)` so callers can override.

```tsx
import { cn } from "@/lib/utils"

<div className={cn("flex items-center gap-2", className)} />
```

## Use Theme Tokens, Not Raw Colors

The design tokens are defined in `app/globals.css` (OKLCH, light + dark).
Always reference them through the Tailwind utilities Tailwind v4 generates from
`@theme inline`.

| Use                     | Token classes                                    |
| ----------------------- | ------------------------------------------------ |
| Surfaces                | `bg-background`, `bg-card`, `bg-muted`, `bg-popover` |
| Text                    | `text-foreground`, `text-muted-foreground`       |
| Brand                   | `bg-primary text-primary-foreground`             |
| Destructive             | `bg-destructive/10 text-destructive`             |
| Borders / rings         | `border-border`, `ring-ring`, `outline-ring/50`  |
| Sidebar surfaces        | `bg-sidebar`, `text-sidebar-foreground`, `border-sidebar-border` |

```tsx
// ❌ BAD
<div className="bg-zinc-900 text-white border-gray-200" />

// ✅ GOOD
<div className="bg-background text-foreground border border-border" />
```

Dark mode is opt-in via the `.dark` class on `<html>` (driven by `next-themes`).
Use the `dark:` variant only for tweaks the tokens cannot express.

## Visual Density (match `Demo` / `FormLayout`)

- Default body type: `text-xs` / `text-xs/relaxed`. Headings use
  `font-heading` and stay small (`text-base`–`text-xl`).
- `Card` defaults to `rounded-none` + `ring-1 ring-foreground/10`. Use
  `<Card size="sm">` for tighter padding inside grids.
- `Item variant="outline"` is the canonical row in lists (see review queues,
  module lists, task lists).
- Buttons use `size="sm"` for in-card actions and `size="xs"` for inline
  actions inside list rows. Reserve the default size for primary page CTAs.
- Form inputs come from `@/components/ui/field` + `@/components/ui/input` /
  `input-group` / `select` / `textarea`. Wrap inputs in `<Field>` with
  `<FieldLabel>` and (when helpful) `<FieldDescription>`.

## Variants — `cva`

For components with multiple visual states, define a `cva()` near the top of
the file (see `components/ui/button.tsx`). Expose `data-variant` / `data-size`
attributes when consumers may want to style descendants.

## shadcn / Radix

- All primitives live in `components/ui/*` and were generated by shadcn
  (`style: radix-lyra`, `baseColor: neutral`). To add a new primitive run:

```bash
npx shadcn@latest add <component>
```

- **Do not** rewrite a primitive to add app-specific behavior. Instead create a
  wrapper in `components/<area>/` (e.g. `components/navbar/app-navbar.tsx`).
- Keep the `data-slot`, `data-variant`, `asChild` patterns intact when
  extending.

## Layouts

| Component         | Where it lives                          | Use it for                       |
| ----------------- | --------------------------------------- | -------------------------------- |
| `NavbarLayout`    | `components/layout/navbar-layout.tsx`   | Authed admin / student tenants   |
| `MainLayout`      | `components/layout/main-layout.tsx`     | Sidebar-driven feature spaces    |
| `FormLayout`      | `components/layout/form-layout.tsx`     | Forms with sticky save bar       |
| `AuthLayout`      | `components/layout/auth-layout.tsx`     | `auth/login`, `auth/forgot`, …   |

`PageHeader` (`components/admin/page-header.tsx`) is the canonical block for
title + description + right-aligned actions. Use it on every page that doesn't
already get a header from the layout shell.

```tsx
<PageHeader
  title="Students"
  description="Create accounts, assign courses, and track each student's path."
  actions={<Button size="sm">…</Button>}
/>
```

## Icons

- Only `@tabler/icons-react`, imported as `Icon*` (e.g. `IconLoader`,
  `IconChevronUp`). Standard sizes: `size-3`, `size-3.5`, `size-4`, `size-5`.
- For icon-only buttons use `<Button size="icon" />` / `icon-sm` / `icon-xs`.
- Pass icons as components (not JSX) when configuring data-driven menus
  (see `MenuNavbarItem.icon`, `MenuSidebarItem.icon`).

## CSS Modules

Use a CSS module **only** for component-local keyframes / animations that
don't fit into Tailwind utilities (see `components/loading/robo-loader.module.css`).
Co-locate the file next to the component and import as `styles`. Continue to
respect `prefers-reduced-motion`.

## Sticky / Scroll Containers

The `min-h-0 min-w-0 flex-1 overflow-y-auto` pattern (see `FormLayout` and
`NavbarLayout`) keeps the body scrollable while the navbar / breadcrumb /
footer stay sticky. Preserve it when editing scroll containers.

## border radius 

Use border radius rounded-md for almost every cards and buttons.
