# Submissions Grading Section Overview

The Submissions section serves as the grading and review center for all student assignments. It handles student-submitted work, providing evaluation workflows and feedback collection.

---

## Key Features

### 1. Review Queue (Inbox)
*   **Unified Queue:** Lists all student submissions waiting for review.
*   **Search & Filter:** Filters submissions by:
    *   **Status:** `Pending` (needs review), `Approved` (graded/passed), `Rejected` (revision requested).
    *   **Course:** Filters by specific courses.
    *   **Search:** Locate specific students or assignments.
*   **List Details:** Shows student name, assignment title, course, and submission timestamp.

### 2. Submission Details Drawer / Modal
Clicking on any submission opens a detailed evaluation sheet:
*   **Student Work Viewer:**
    *   Renders the text response submitted by the student.
    *   Provides links to download or view attached files/receipts.
*   **Grading Controls:**
    *   **Approve:** Marks the task as completed. This increments student module progress and unlocks the next sequence in the curriculum.
    *   **Reject (Request Revision):** Sends the task back to the student. The status changes to `Rejected`, notifying the student to rework and resubmit.
*   **Admin Feedback:** A text box allowing the evaluator to add detailed feedback, change requests, or praise.
